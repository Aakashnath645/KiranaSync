# Security Policy

This document outlines the security policy for KiranaSync. We take security seriously and appreciate proactive reporting of any vulnerabilities.

## Supported Versions

Only the latest release version of KiranaSync receives security updates. If you identify a security concern, please ensure you are testing against the most recent build.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | Yes (Current)      |
| < 1.0   | No                 |

## Reporting a Vulnerability

If you discover a security vulnerability in KiranaSync, please do not disclose it publicly or open a public issue. Instead, report it through the private channel described below:

1. **Email Communication**: Report all security findings to our security contact at aakashnath645@gmail.com.
2. **Details to Include**: Provide a comprehensive description of the vulnerability, including:
   - Steps to reproduce or a proof-of-concept.
   - Impact assessment (local state exploitation, permission bypass, etc.).
   - Proposed remediation steps or patch suggestions.

## Disclosure Policy

Once a vulnerability report is received, we will:
1. Acknowledge receipt of the report within 48 hours.
2. Investigate the vulnerability and determine severity.
3. Keep the reporter updated as a fix is developed.
4. Maintain confidentiality of both the reporter and vulnerability until a patch or mitigation is publicly released.

Thank you for helping keep KiranaSync secure.
